# ScyllaDB 高效能 Go 專案範例 (test_scylla)

本專案展示了如何使用 Go 語言配合 ScyllaDB 建立一個具備高可用性（High Availability）與高效能的 RESTful API。採用了 3 個節點的 ScyllaDB 叢集架構，並使用 `gocqlx` 驅動程式以發揮最佳效能。

## 🚀 技術棧

- **Backend**: Go 1.25+ (Gin Framework)
- **Database**: ScyllaDB 6.0.2 (3-node cluster)
- **Driver**: `scylladb/gocqlx/v2` (支援 Token-aware 負載平衡與高效能對照)
- **Infrastructure**: Docker Compose

## 🛠 快速開始

### 1. 啟動 ScyllaDB 叢集
使用 Docker Compose 啟動 3 個 ScyllaDB 節點與自動初始化容器：

```bash
docker compose up -d
```

> **注意**：初次啟動叢集需要 1~2 分鐘進行同步。您可以使用以下指令檢查狀態，直到看到 3 個節點都顯示為 `UN` (Up/Normal)：
> ```bash
> docker exec -it scylla-1 nodetool status
> ```

### 2. 啟動 Go API 伺服器
確保本地已安裝 Go 環境，然後執行：

```bash
go run cmd/server/main.go
```

伺服器將啟動於 `http://localhost:8080`。

### 3. 執行功能測試
在另一個終端機執行自動化測試腳本：

```bash
chmod +x scripts/test_api.sh
./scripts/test_api.sh
```

## ⚙️ ScyllaDB 優化重點

- **Token-aware Balancing**: 連線池會自動根據資料分佈（Token）直接連線至負責該資料的節點，減少節點間的額外轉發。
- **3-Node Cluster**: 提供容錯能力。即使其中一個節點故障，系統仍可透過其餘節點正常運作。
- **Developer Mode**: 在開發環境中啟動 `--developer-mode 1` 以跳過硬體限制檢查，加速啟動。
- **gocqlx**: 使用 `gocqlx` 簡化了 SQL 語法拼湊，並提供比標準 `gocql` 更高的資料綁定效能。

## 🛠 常用維護指令

- **檢查叢集狀態**: `docker exec -it scylla-1 nodetool status`
- **進入 CQL 終端**: `docker exec -it scylla-1 cqlsh`
- **查看容器日誌**: `docker compose logs -f scylla-1`
